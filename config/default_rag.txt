## First-Run Setup Knowledge
- On first open, MNI creates database tables automatically and opens the admin setup/login flow.
- Default first-time unlock is username `admin` and password `admin@123@321`.
- After first unlock, the owner should set their permanent login method: password, Google login, or password plus Google.
- Local setup can run `python scripts/setup_local.py --install-deps --install-node` to create `.env`, Google auth config, install dependencies, and initialize the database.
- Secrets should stay in `.env`, ignored config files, the database secret store, or deployment environment variables.

## Google Meet Setup
- Google Meet automation needs Google OAuth Client ID, Client Secret, and Refresh Token.
- The refresh token must include permission to create Meet spaces, commonly `https://www.googleapis.com/auth/meetings.space.created`.
- In Google Cloud Console, create/select a project, configure OAuth consent, create OAuth Client credentials, and add the correct redirect/origin for the deployment.
- In MNI, enter these values in `Connect Apps` under `Google Meet Automation`, or use the ignored local file `config/google_auth.json`.
- If Meet fails, check client ID, client secret, refresh token, Google API access, OAuth scopes, and whether the deployment URL matches Google Cloud settings.

## Email Setup
- Email automation sends mail through SMTP.
- The owner must add SMTP host, port, username, password/app password, and From email in `Connect Apps` under `Email Automation`.
- Gmail usually requires an App Password, not the normal Gmail password.
- Emails are logged in the Emails panel with sent/failed status.
- If email fails, check SMTP credentials, port, TLS policy, provider app-password rules, and recipient formatting.

## n8n Automation Setup
- n8n is optional and user-owned: self-hosted n8n or n8n Cloud both work.
- Save n8n Base URL, API Key, Default Webhook URL, and Bot Trigger Keyword in the `Automations` panel.
- MNI can send a test trigger from the admin panel or trigger n8n from the MNI Chat dropdown.
- A workflow can save leads, send emails, create Google Sheets rows, create CRM records, generate tickets, create Google Meet links, or return a response to chat.
- If n8n fails, check webhook URL, API key, workflow active status, network access, and payload expectations.

## Telegram Setup
- For local development, MNI uses Telegram long polling. Public webhooks are only useful when the backend has a public HTTPS URL.
- If a stale Telegram webhook exists while running locally, MNI removes it so polling can receive messages.
- In private chat, the bot replies directly. In groups, mention the bot or reply to the bot unless group auto-reply rules are changed.
- If Telegram does not reply, check token validity, `telegram_enabled`, worker mode, polling logs, and whether the bot is being mentioned in groups.
