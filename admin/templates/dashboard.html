Architecture
How this bot stack fits together
MNI uses a Flask backend as the brain and admin API, a server-rendered admin GUI, SQLite/Postgres for runtime data, and optional Node workers for Telegram and Discord. Most panels on this screen call real backend endpoints under /api/admin.
Local setup
Run python scripts/setup_local.py --install-deps --install-node once. It creates .env, creates config/google_auth.json, prepares instance/, and initializes the database tables.
Start the backend with python backend/app.py. Start bot workers separately with node bot/index.js when platform tokens are configured.
Backend Map
Flask app
backend/app.py registers auth, admin, chat, and platform webhook routes.
Admin API
backend/routes/admin.py powers this GUI: config, RAG, APIs, users, bots, outreach, logs, and monitor data.
Chat API
/api/chat/message runs messages through moderation, config, local data, AI routing, and logging.
Webhooks
backend/routes/platforms.py receives Telegram, Discord, WhatsApp, and Instagram events.
Frontend Map
Login
admin/templates/login.html handles first admin registration and password login.
Dashboard
admin/templates/dashboard.html is the connected admin console. Panels are hidden/shown in-place and fetch JSON from the backend.
Secrets
Sensitive fields are saved encrypted in config rows when supported. Real local secret files are ignored by git.
Dead pages
Disconnected standalone demo pages were removed so the GUI only exposes working backend-connected tools.
Database And Data
Default DB
Local runs use the SQLite database in instance/ unless DATABASE_URL is set.
Hosted DB
Render/Railway/Postgres should use DATABASE_URL. The Database Deployment card can save a future external URL for the next restart.
Auto seed
On startup the app creates tables, seeds default config, local data sources, and default data agents.
Cleanup
Generated database, logs, and Python caches are ignored and should not be committed.
Google Meet Setup
Option 1
Fill Google Client ID, Client Secret, and Refresh Token in Platform Integration, then save.
Option 2
Copy config/google_auth.example.json to config/google_auth.json and fill OAuth values there.
Scope
Use https://www.googleapis.com/auth/meetings.space.created for creating Meet spaces.
Usage
After setup, ask the bot to create or schedule a Google Meet. The utility handler calls the Meet REST API and returns the link.
